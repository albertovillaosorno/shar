# Decision title

Read the [SHAR Documentation Guide](../README.md) before instantiating this
record. Replace every instruction with finished decision prose.

- Status: Proposed
- Decision date: YYYY-MM-DD
- Scope: Concise repository concern

## Context

State the repository problem, constraint, and evidence that require one durable
choice. Do not include a technical walkthrough, user workstation preference,
external format explanation, command, or concrete repository path.

## Decision

State one repository-impacting choice in normative language. Keep the decision
independent of file layout and local environment.

## Consequences

Describe required behavior, trade-offs, migration effects, and validation
expectations.

## Rejected alternatives

List materially considered alternatives and why they were rejected.

Add a `Supersession` section only when this record replaces a current decision;
name the prior decision title and update every repository reference. Do not add
an empty no-supersession section. When a document is technical knowledge rather
than a decision, move it to the technical catalog instead of keeping a
placeholder ADR.
