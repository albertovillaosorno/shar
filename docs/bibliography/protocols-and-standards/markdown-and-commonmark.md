This non-governing record documents a plain-text markup family without granting
rights in authored prose, linked material, embedded media, or rendered outputs.

## Covered Material

Markdown files authored or generated by SHAR, including headings, paragraphs,
lists, code spans and blocks, links, images, tables as supported by tooling,
reference definitions, and file-local lint directives.

## Repository Use And Scope

SHAR uses Markdown for public documentation, architecture decisions, legal
research, bibliography records, and RTF conversion output. The canonical
Markdownlint configuration defines repository style, while CommonMark provides a
published syntax baseline. Tool-specific extensions must be identified rather
than treated as universal Markdown.

## Provenance And Version History

Markdown originated as a text-to-HTML syntax and later developed multiple
dialects. CommonMark publishes a versioned specification and test suite intended
to reduce ambiguity. GitHub Flavored Markdown and other extensions are separate
profiles and must not be assumed unless the relevant renderer or validator is
identified.

## License Or Terms Basis

Use of Markdown syntax does not license prose, code excerpts, images, linked
material, or renderer software. CommonMark specifications, reference software,
and extensions have their own license and publication terms.

## Distribution, Modification, And Compatibility

A Markdown file may render differently across engines. Valid markup may still
contain copyrighted text, unsafe links, confidential paths, or misleading
claims. Generated Markdown derived from source documents retains the source
content's provenance and rights issues.

## Compliance Posture

- Identify the renderer, lint profile, and supported extensions where output
  depends on them.
- Keep local lint exceptions rule-specific and justified.
- Validate links, references, headings, and code fences through `validate.sh`.
- Review copied prose, code, images, and generated conversions before
  publication or distribution.
- Do not treat rendering success as legal or factual verification.

## Technical Baseline And SHAR Profile

### Public baseline

CommonMark 0.31.2 is the identified baseline for core Markdown parsing. GitHub
Flavored Markdown is a separate extension profile that adds features including
tables, task-list items, strikethrough, extended autolinks, and restrictions on
selected raw HTML tags.

Neither specification determines SHAR's complete lint policy, repository link
rules, rendered-site behavior, or file-local exceptions.

### SHAR profile

The canonical validator invokes repository-managed `markdownlint-cli2` and the
`markdownlint` rule engine with an explicit configuration and enumerated file
list. Validation policy, rather than a renderer label alone, controls accepted
repository Markdown.

The bibliography template uses brace-delimited prompts instead of unresolved
reference labels, so MD052 remains enabled without a file-wide suppression. Real
links, reference definitions, inline code, and task-list markers retain their
standard Markdown syntax.

### Use-specific evidence limits

Before claiming rendering parity across GitHub or any generated documentation
surface, maintain a renderer matrix, canonical configuration inventory,
extension-specific regression tests, and confirmation that every file-local
exception remains necessary and narrowly scoped.
