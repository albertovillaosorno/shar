# {Legal Subject} Legal Record Template

> The canonical [Legal Research Disclaimer](disclaimer.md) governs this template
> and every record instantiated from it.

Read the [SHAR Documentation Guide](../README.md) before instantiating this
record. Replace every braced instruction with finished legal-research prose.

Every instantiated record must link the canonical **Legal Research Disclaimer**
using the correct relative path for its category. Do not copy, paraphrase, or
vary the disclaimer inside the record.

Select one record family before replacing the template instructions. Every legal
record uses this common baseline:

- Status: {Descriptive verification and factual-application boundary.}
- As-of date: {YYYY-MM-DD.}
- Counsel review: {Not performed | Internal review | Counsel reviewed}.

Add the fields required by the selected family:

- Case record: `Court`, `Authority`, and `Decision date`; add `Docket`, `Panel`,
  or later-history fields only when material.
- Statute, regulation, doctrine, contract, license, platform, or jurisdiction
  record: `Jurisdiction` and `Authority level`; add currentness metadata when it
  identifies a separately verified proposition.
- Repository-boundary record: identify its repository purpose and legal-review
  boundary without presenting repository policy as external legal authority.

Do not mix unlike metadata models merely to make records appear uniform.

Authority-analysis records normally use the sections below. A
repository-boundary record may instead use `Purpose`, `Repository Boundary`,
`Legal Review Boundary`, `Sources`, `Validation`, and `Stop Conditions` when
that shape states the actual repository function more accurately.

## Verification Checklist

- Verify current text and effective date.
- Read the complete authority, including dissents and procedural posture.
- Identify controlling jurisdiction and later treatment.
- Separate copyright, contract, anti-circumvention, trademark, patent, trade
  secret, and platform-policy questions.
- Obtain qualified counsel before a consequential conclusion.
